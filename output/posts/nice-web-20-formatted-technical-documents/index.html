The other day I randomly came across this nice youtube video. It was a suggestion of youtube, so probably it is starting to get to know me too well.

In any case the guy wrote a very nice CSS styleguide for technical documentation that you can use when creating some technical documentation.

You can then simply code in orgmode and by specifying the location of the guy CSS style when exporting your org mode document into html it will be nicely rendered and you will have then the opportunity to nicely write your technical documentation.


Set-up

In order to do this you can check out the repo with the specified CSS styleguide.

You can then clone it. Then by simply inserting the following header 

#+TITLE: IBM Blockchain Fabric
#+AUTHOR:    Marco Hassan 
#+DATE:      2020-02-12
#+LANGUAGE:  en
#+OPTIONS:   H:4 num:nil toc:2
#+SETUPFILE: ~/Desktop/TechnicalPosts/org-html-themes/setup/theme-readtheorg.setup

-----

specifying in the setupfile the path to our friend style files you will end up exporting in the nice documentation format.

I experimented with it for a demo on Blockchain I had to recently run. I created a github-page for this.

You can do that simply by creating a new repository in Github and by specifying in the settings that you want the repository to act as a Github Page. You will then be given an URL where your HTML page will be rendered. Following this link and entering the path to the HTML file you want to display you will see it in the nicely formatted style.

You can have an idea of it - if I did not remove the repo yet - following this link.

Side Note: I had to select a Theme and then to empty the .yml for the page to be rendered. Not sure why it did not work in the first place. Maybe I was to impatient.
